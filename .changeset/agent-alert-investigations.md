feat(agent): trigger an on-call agent investigation when an alert fires. On the OK/PENDING -> ALERT edge (once per fire, not per breaching tick), the alert task marks the AlertHistory record and the alert provider dispatches a Bearer-authenticated request to the agent's `investigateAlert` workflow. The agent investigates via its read-only MCP tools and posts a findings summary back to a team-scoped, credential-authenticated internal endpoint (`POST /agent/investigations`), which stores it on the AlertHistory record. Cost controls are server-side: an atomic per-alert claim enforces at most one investigation per alert per hour, grouped alerts cap at 5 dispatches per evaluation, and each task run caps at 10 across all alerts. Summaries are served by the new `GET /alerts/investigations` endpoint. Gated by `AGENT_INVESTIGATIONS_ENABLED` (default off); delivery is at-most-once by design. Also mounts the MCP server with an explicit allowed-hosts list (localhost plus the Compose service name and `MCP_ALLOWED_HOSTS`) so in-network MCP clients work without disabling DNS-rebinding protection.
